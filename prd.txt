1. Overview
Product Summary

Anchored is a cross-platform civic utility app designed to help Halifax residents manage municipal services and stay informed about local civic issues. The app centralizes fragmented municipal information—such as garbage collection schedules, street closures, and civic issue data—into a personalized, location-aware mobile experience.

Problem Statement

Municipal service information in Halifax is currently spread across multiple sources (e.g., open data portals and the 311 system). Residents must manually search for updates and receive little to no personalization. This results in:

Missed waste collection and service dates

Low awareness of nearby civic disruptions/issues

No easy way to track relevant local issues in one place

Value Proposition

Anchored provides a single, personalized civic dashboard for Halifax residents by combining:

Service reminders (especially waste collection)

Nearby civic issue awareness

Saved issue tracking

Location-aware map visualization

Push notifications and native integrations

6. Product Requirements
6.1 Functional Requirements
A. Authentication Module

Description: Users must authenticate via email and password.

Requirements

Users can register with email/password

Users can log in/log out

Users can remain signed in across sessions

Authentication managed with Supabase Auth

Password reset flow (recommended for MVP if time permits)

Acceptance Criteria

New user can create account and access app

Existing user can log in and return to dashboard

Invalid credentials return clear error message

B. User Profile & Settings

Description: Users can manage address, notifications, and permissions.

Requirements

User can enter/update primary address

User can manage notification settings (on/off)

User can manage permission prompts (location, camera, notifications)

Optional theme/UI preferences storage

Acceptance Criteria

Address is saved and used to personalize service/reminder data

Notification toggle persists after app restart

Permission status is reflected in settings UI

C. Service Reminders (Core MVP Feature)

Description: Personalized reminders for waste collection and municipal services.

Requirements

App fetches waste/service schedule based on user’s address/service zone

User can view upcoming service schedule

User can enable/disable reminders

App schedules push notifications for upcoming services

Reminder timing configurable (e.g., night before / morning of) if time permits

Acceptance Criteria

User sees next service dates on Home screen

Notifications are scheduled correctly based on service data

Changing address updates schedule and reminders

D. Civic Issues Awareness

Description: Users can view nearby civic issues and disruptions.

Requirements

App fetches civic issue/open-data records (e.g., closures, construction)

Issues are filterable by proximity to user (location-based radius)

Issues can be displayed in list and/or map marker format

Users can save/bookmark issues for tracking (if included in MVP)

Acceptance Criteria

Nearby issues load using current location or address

User can tap an issue marker to view details

Basic filtering (distance/type if implemented) updates results correctly

E. Map Integration

Description: Interactive map showing user location and civic issue markers.

Requirements

Use platform-appropriate map integration:

Google Maps (Android)

Apple Maps (iOS) / map abstraction via React Native library

Display user’s live location (with permission)

Display civic issues as custom markers

Marker tap opens issue details

Filter displayed issues by radius from user

Acceptance Criteria

Map loads on both iOS and Android

User location displays when permission granted

Marker selection opens correct issue detail view

F. Issue Details Screen

Description: Detailed view for a selected civic issue.

Requirements

Show issue title/type

Show status (if available)

Show location/address/coordinates (if available)

Show description and timestamp/update time (if available)

Link out to official source / 311 page where applicable

Acceptance Criteria

Selected marker/list item loads the correct issue details

Missing fields are handled gracefully (no crashes)

G. Issue Reporting Photo Flow (Handoff Flow)

Description: User can capture a photo and be redirected to the official 311 site.

Requirements

Map screen includes “+” action

User is prompted to open camera

User captures photo

Photo is temporarily saved locally

App redirects user to Halifax 311 website for issue reporting completion

Acceptance Criteria

Camera opens successfully with permission

Captured photo is temporarily stored

Redirect to 311 webpage succeeds

6.2 Non-Functional Requirements
Performance

Initial screen load should be responsive on typical mobile devices

Cached civic/service data should reduce unnecessary repeated requests

Map interactions should remain smooth with moderate marker counts

Reliability

App should handle API failures gracefully with fallback messaging

Notification scheduling should be robust to app restarts (where platform behavior allows)

Security & Privacy

Secure auth via Supabase

Store sensitive data in secure storage where appropriate (tokens/session data)

Location/camera permissions requested transparently and only when needed

Only collect data necessary for app functionality

Usability

Simple onboarding and settings flow

Clear reminders and actionable home dashboard

Readable map markers and issue details

Cross-Platform Compatibility

Consistent core experience across iOS and Android

Account for map SDK/platform differences and notification behavior differences

13. Key Technical Challenges & Risks
1) Frequently Updated Civic Data

Risk: Data freshness, schema changes, rate limits, inconsistent update cadence
Mitigation: Caching + refresh strategy, defensive parsing, timestamp display, retry/error handling

2) Location-Based Filtering Accuracy

Risk: Inaccurate coordinates, permission denial, geocoding/address mismatch
Mitigation: Fallback to saved address, configurable radius, clear permission UX, validation of coordinate data

3) Cross-Platform Map Behavior Differences

Risk: Marker rendering/interaction inconsistencies between iOS and Android
Mitigation: Use consistent map abstraction library, test on both platforms early, simplify marker interactions

4) Notification Scheduling from Time-Based Data

Risk: Time zone handling, platform scheduling differences, stale schedules after updates
Mitigation: Store normalized event times, reschedule on data refresh/address change, test edge cases and DST
